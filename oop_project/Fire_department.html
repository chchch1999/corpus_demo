Fires Department Resource Management

Overview

The fire package is designed to provide resource management capabilities for a fire department. It includes six modules that handle user management, vehicle management, and various resource management operations. This documentation will provide an overview of each module and how to use the interactive page to manage resources.

Project Structure

The project is structured as follows:

    Fire_department/        
    ├── fire/               
    │   ├── manager.py      
    │   ├── test.py         
    │   ├── vehicle.py      
    │   ├── vehicle_menu.py 
    │   ├── option.py       
    │   └── user.py         
    ├── manage_department.ipynb 
    ├── users.json          
    └── vehicles.json

Interactive Page - manage_department.ipynb

Overview

The interactive page manage_department.ipynb serves as the user interface for managing resources in the Fires Department Resource Management package. Users can run various operations, manage resources, and execute unit tests using this interactive page.

Instructions

  1. Getting Started: Upon launching the interactive page, you'll see information about your home directory and the current working directory. This provides context about the environment.
  2. Directory Contents: The script lists the contents of the current directory, allowing you to see available files and the package fire.
  3. Resource Management: The page imports the manager module and runs its main() function. This function initializes the resource management system, allowing users to perform resource-related operations.
  4. Unit Testing: The page also performs unit tests by importing the test module, which contains tests for the vehicle management module. It checks if the tests were successful and exits with a status code of 1 if they fail.

Attention!

*Restart the terminal everytime after you run the cell: To prevent the "call_sign must be unique" error, it is essential to restart the terminal before running the interactive page (manage_department.ipynb). This ensures a clean environment and prevents conflicts in the vehicle management module.

Module 1 - manager.py

Overview

The manager.py module is the core of the Fires Department Resource Management package. It manages user authentication, resource registration, and provides an interactive menu for users to perform various resource-related operations.

Key Features

  • User authentication and registration.
  • Resource registration, including complaints, memos, and action reports.
  • User-friendly menu for accessing functions.

Module 2 - test.py

Overview

The test.py module contains unit tests for the vehicle management module. It ensures the correctness of vehicle-related functions and properties.

Key Features

  • Unit tests for vehicle properties and methods.
  • Ensures the integrity of the vehicle management module.

Module 3 - vehicle.py

Overview

The vehicle.py module defines the Vehicle class, which represents vehicles used by the fire department. It includes properties and methods for managing vehicle data.

Key Features

  • Definition of the Vehicle class with properties for call sign, purpose, weight, fuel level, and range.
  • Ensures data integrity and uniqueness of call signs.

Module 4 - vehicle_menu.py

Overview

The vehicle_menu.py module provides a menu system for managing vehicles. Users can create, update, and view information about vehicles through this module.

Key Features

  • Vehicle management menu.
  • Functions for creating, updating, and viewing vehicle information.
  • Data persistence through JSON files.

Module 5 - option.py

Overview

The option.py module manages menu options and provides a framework for handling user choices. It includes classes for defining menu options, safe options with logging, and report-related options.

Key Features

  • Option classes for defining menu choices.
  • SafeOption class for secure menu options with logging.
  • Report and RegisteredReport classes for handling resource reports.

Module 6 - user.py

Overview

The user.py module handles user management and authentication. It defines the User class and SecureUser subclass for secure user registration and login.

Key Features

  • User management and authentication.
  • Secure user registration with password requirements.
  • Secure user login and logout functionality.
